# QuickTool 愿景文档

## 一、项目定位

QuickTool 是一个常驻在 Windows 系统托盘（任务栏右下角）的轻量级个人效率工具。它的核心目标是把「平时零散写的脚本」变成「随时一点即用的工具集」，让写过的每一个脚本都能被方便地调用，而不是躺在某个文件夹里被遗忘。

一句话概括：**用最小的心智负担，把分散的脚本聚合成一个随身工具箱。**

## 二、核心理念

1. **零配置**：把脚本扔进指定目录，UI 自动出现一个入口，不需要改代码、不需要编辑配置文件。
2. **常驻不打扰**：默认最小化到系统托盘，只有需要用的时候才弹出；用完消失，不占据桌面空间。
3. **可生长**：功能按模块演进，先把「脚本启动器」做稳，再逐步加入其它常用小工具。
4. **本地优先**：所有脚本、配置、日志都在本地，不依赖网络，不上传任何数据。

## 三、首期功能（MVP）

### 3.1 系统托盘常驻

- 启动后不显示主窗口，只在任务栏右下角出现一个小图标。
- 左键单击图标：弹出脚本面板（靠近图标位置，类似网易云音乐/TIM 的迷你面板）。
- 右键单击图标：弹出菜单，包含「打开脚本目录」「刷新」「设置」「开机自启」「退出」等。
- 双击图标：打开主窗口（完整视图，便于管理）。

### 3.2 脚本自动发现（核心）

- 用户在「设置」中指定一个脚本根目录（默认 `./scripts`）。
- 程序递归扫描该目录，把每一个可执行脚本渲染成一个按钮/卡片。
- **支持的脚本类型**（首期）：
  - `.ps1`  PowerShell
  - `.bat` / `.cmd`  批处理
  - `.py`  Python（调用系统 Python 或虚拟环境）
  - `.exe`  可执行文件
  - `.js` / `.ts`  Node 脚本（可选）
- **自动感知**：使用文件系统监听（`FileSystemWatcher` / `chokidar` / `watchdog`），脚本新增、删除、重命名时 UI 实时刷新，无需重启。

### 3.3 脚本元信息（可选增强）

每个脚本可以通过「同名 `.meta.json`」或「脚本文件头部注释」提供元信息，用来美化 UI：

```json
{
  "name": "清理临时文件",
  "description": "删除 %TEMP% 下 7 天前的文件",
  "icon": "broom",
  "category": "系统维护",
  "args": ["--dry-run"],
  "runAs": "admin",
  "hidden": false
}
```

如果没有提供，程序会根据文件名自动生成显示名（例如 `clean-temp.ps1` → `Clean Temp`）。

### 3.4 运行与反馈

- 点击按钮：在后台启动脚本，托盘图标短暂高亮或显示气泡提示。
- 运行日志：可在主窗口「运行历史」里查看每次运行的 stdout / stderr / 退出码 / 耗时。
- 失败提示：脚本非 0 退出码时，弹出轻量通知（不打断操作）。
- 并发控制：允许同时运行多个脚本；正在运行的脚本在 UI 上有明显标识。

### 3.5 基础设置

- 脚本目录路径
- 主题（浅色 / 深色 / 跟随系统）
- 开机自启
- 面板快捷键（例如 `Ctrl+Alt+Q` 唤出面板）
- 日志保留条数

支持：

- 收藏/置顶常用脚本
- 拖拽排序（可选）

## 五、技术选型（已确定）

**Electron**（主进程 Node.js + 渲染进程 HTML/CSS/JS）。

- 渲染层首期使用原生 HTML/CSS/JS，避免引入构建工具，保证「拉下来就能跑」。
- 主进程使用 CommonJS 脚本（无 TypeScript 编译步骤）。
- 文件监听：[chokidar](https://github.com/paulmillr/chokidar)。
- 后续若 UI 复杂度上升，再按需引入 Vite + React/TS，迁移成本可控。
- 打包：后续接入 [electron-builder](https://www.electron.build/)，输出便携版 + 安装包。

**脚本目录位置**：固定为应用根目录下的 `./scripts/`（即与 `package.json` 同级），方便用户直接打开、编辑、用 Git 管理。后续在「设置」中允许覆盖。

## 六、目录结构（规划）

```
QuickTool/
├── VISION.md                 # 本文档
├── README.md                 # 使用说明（后续补充）
├── src/                      # 应用源码
├── scripts/                  # 默认脚本目录（用户可修改）
│   ├── 示例-hello.ps1
│   └── 示例-hello.meta.json
├── assets/                   # 图标、主题资源
├── docs/                     # 设计文档、开发笔记
└── dist/                     # 打包产物
```

## 七、后续可扩展方向（非 MVP）

- **剪贴板历史**：托盘点击快速回填。
- **常用文件/文件夹快捷跳转**。
- **HTTP / 命令片段库**：把常用 curl、adb、docker 命令也纳入启动器。
- **快捷键绑定**：给特定脚本绑定全局热键，不打开面板也能触发。
- **参数交互**：脚本可声明参数，运行前弹出表单让用户填写。
- **脚本市场/同步**：把自己的脚本目录同步到 GitHub Gist / 私有仓库，多台机器共享。
- **插件机制**：第三方用 JS/Python 写扩展面板。

## 八、非目标（明确不做）

- 不做重量级 IDE / 编辑器功能，脚本编辑请用用户自己喜欢的编辑器。
- 不做团队协作、不做云同步后端服务（可能通过 Git 同步，但不自建服务）。
- 不做跨平台优先设计（优先 Windows，Linux/macOS 视情况而定）。

## 九、里程碑

- **M0 设计确认**：完成本文档 + 技术选型敲定。
- **M1 骨架跑通**：系统托盘 + 主窗口 + 读取一个写死目录并列出脚本 + 点击可运行。
- **M2 自动刷新**：文件监听，增删改实时反映到 UI。
- **M3 元信息与分组**：支持 `.meta.json`、分组、图标、搜索。
- **M4 体验打磨**：运行历史、设置页、开机自启、主题、快捷键。
- **M5 打包发布**：单文件安装/便携包，首个 Release。
